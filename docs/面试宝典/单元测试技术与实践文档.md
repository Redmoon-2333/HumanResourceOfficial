## 1. 单元测试基础概念

### 1.1 什么是单元测试？

**定义**: 对软件中最小可测试单元（通常是方法或函数）进行检查和验证。

**核心原则**:
- ✅ **独立性**: 每个测试用例相互独立，不依赖执行顺序
- ✅ **可重复性**: 无论运行多少次，结果一致
- ✅ **原子性**: 一个测试只验证一个行为
- ✅ **快速执行**: 测试应在毫秒级完成

### 1.2 为什么需要单元测试？

| 好处 | 说明 |
|------|------|
| 🐛 **早期发现 Bug** | 在代码提交前发现问题，修复成本最低 |
| 📝 **活文档** | 测试用例是最好的代码行为文档 |
| 🔧 **重构信心** | 有测试保护，重构时不怕破坏现有功能 |
| 📊 **质量指标** | 测试覆盖率反映代码质量 |
| 💡 **设计改进** | 写测试时发现设计问题（如依赖过多） |

### 1.3 测试金字塔

```
        /\
       /  \
      / E2E \      端到端测试 (10%)
     /--------\    慢、昂贵、易碎
    /          \
   / Integration\  集成测试 (20%)
  /--------------\  中等速度、验证模块协作
 /                \
/    Unit Tests    \ 单元测试 (70%)
--------------------  快、便宜、稳定
```

**本项目策略**: 以单元测试为主，逐步补充集成测试

## 3. 如何编写单元测试

### 3.1 AAA 模式 (核心范式)

每个测试用例遵循 **Arrange-Act-Assert** 三段式:

```java
@Test
@DisplayName("根据 ID 获取活动 - 成功")
void getActivityById_Success() {
    // ========== Arrange (准备) ==========
    // 准备测试数据和 Mock 行为
    when(activityMapper.selectById(1)).thenReturn(testActivity);
    
    // ========== Act (执行) ==========
    // 执行被测试的方法
    Activity result = activityService.getActivityById(1);
    
    // ========== Assert (断言) ==========
    // 验证结果是否符合预期
    assertNotNull(result);
    assertEquals(1, result.getActivityId().intValue());
    verify(activityMapper, times(1)).selectById(1);
}
```

### 3.2 注解速查表

| 注解 | 作用 | 示例 |
|------|------|------|
| `@Test` | 标记测试方法 | `@Test` |
| `@DisplayName` | 测试名称 (支持中文) | `@DisplayName("登录 - 成功")` |
| `@BeforeEach` | 每个测试前执行 | `setUp()` 准备数据 |
| `@ExtendWith` | 扩展配置 | `@ExtendWith(MockitoExtension.class)` |
| `@Mock` | 创建 Mock 对象 | `@Mock private ActivityMapper mapper;` |
| `@InjectMocks` | 注入 Mock 到被测对象 | `@InjectMocks private ActivityService service;` |
| `@WebMvcTest` | Web 层切片测试 | `@WebMvcTest(AuthController.class)` |

### 3.3 Mock 基础

**什么是 Mock?**
- Mock 是模拟真实对象的"假"对象
- 可以预设行为 (`when().thenReturn()`)
- 可以验证调用 (`verify()`)

**常用 Mock 操作**:

```java
// 1. 预设返回值
when(mapper.selectById(1)).thenReturn(testEntity);

// 2. 预设异常
when(service.doSomething()).thenThrow(new BusinessException());

// 3. 无返回值 (void 方法)
doNothing().when(util).deleteFile(anyString());

// 4. 验证调用
verify(mapper, times(1)).selectById(1);
verify(service, never()).doSomething(); // 从未调用

// 5. 参数匹配
when(mapper.findByUsername(anyString())).thenReturn(user);
when(mapper.findByIds(anyList())).thenReturn(users);
```

### 3.4 断言方法

| 断言 | 用途 | 示例 |
|------|------|------|
| `assertEquals` | 相等断言 | `assertEquals(1, result.getId())` |
| `assertNotNull` | 非空断言 | `assertNotNull(result)` |
| `assertNull` | 为空断言 | `assertNull(error)` |
| `assertTrue` | 为真断言 | `assertTrue(success)` |
| `assertFalse` | 为假断言 | `assertFalse(failed)` |
| `assertThrows` | 异常断言 | `assertThrows(Exception.class, () -> {...})` |
| `assertAll` | 分组断言 | `assertAll(() -> ..., () -> ...)` |

## 4. 测试用例详解

### 4.1 ActivityServiceTest - 活动服务测试

**测试范围**: 活动 CRUD、图片管理

**核心测试场景**:

| 测试方法 | 测试场景 | 关键点 |
|---------|---------|-------|
| `createActivity_Success` | 创建活动成功 | 验证 Mapper 调用 |
| `getActivityById_Success` | 查询活动成功 | 验证返回结果 |
| `getActivityById_NotFound` | 活动不存在 | 验证异常抛出 |
| `updateActivity_Success` | 更新活动成功 | 验证缓存清除 |
| `deleteActivity_Success` | 删除活动成功 | 验证关联图片删除 |

**代码示例**:

```java
@Test
@DisplayName("删除活动 - 成功")
void deleteActivity_Success() {
    // Arrange
    when(activityMapper.selectById(1)).thenReturn(testActivity);
    List<ActivityImage> images = new ArrayList<>();
    images.add(testImage);
    when(activityImageMapper.findByActivityId(1)).thenReturn(images);
    doNothing().when(localFileUtil).deleteFile(anyString());
    when(activityMapper.deleteById(1)).thenReturn(1);

    // Act
    activityService.deleteActivity(1);

    // Assert
    verify(activityMapper, times(1)).selectById(1);
    verify(activityImageMapper, times(1)).findByActivityId(1);
    verify(localFileUtil, times(1)).deleteFile(anyString());
    verify(activityMapper, times(1)).deleteById(1);
}
```

**核心测试场景**:

**代码示例**:

```java
@Test

    // Arrange
    ActivationCode expiredCode = new ActivationCode();
    expiredCode.setCode("VALID_CODE_123");

    when(userMapper.countByUsername("newuser")).thenReturn(0);

        authService.register(registerRequest);

    assertTrue(exception.getMessage().contains("激活码已过期"));
```

### 4.4 AuthControllerTest - 认证控制器测试

**测试范围**: REST API 端点

**核心测试场景**:

| 测试方法 | 测试场景 | 关键点 |
|---------|---------|-------|
| `login_Success` | 登录 API | 验证 HTTP 响应 |
| `login_ValidationFailed` | 参数验证失败 | 验证 400 响应 |
| `register_Success` | 注册 API | 验证返回数据 |
| `getCurrentUser_Success` | 获取当前用户 | 验证 Authorization 头 |
| `logout_Success` | 退出登录 | 验证 Token 黑名单 |

**代码示例**:

```java
@Test
@DisplayName("用户登录 - 成功")
void login_Success() throws Exception {
    // Arrange
    LoginRequest loginRequest = new LoginRequest();

            .andExpect(jsonPath("$.data.token").value("valid_token"));
```

## 5. 最佳实践与常见陷阱

### 5.1 命名规范

**✅ 推荐**:
```java
@Test
@DisplayName("创建活动 - 成功")
void createActivity_Success() { ... }

@Test
@DisplayName("根据 ID 获取活动 - 不存在")
void getActivityById_NotFound() { ... }
```

**❌ 避免**:
```java
@Test
void test1() { ... }  // 无意义命名

@Test
@DisplayName("测试")
void testCreate() { ... }  // 过于笼统
```

### 5.2 测试独立性

**✅ 推荐**:
```java
@BeforeEach
void setUp() {
    // 每个测试都重新准备数据
    testActivity = new Activity();
    testActivity.setActivityId(1);
    testActivity.setActivityName("测试活动");
}
```

**❌ 避免**:
```java
// 测试之间共享可变状态
private static int counter = 0;  // 危险！

@Test
void test1() { counter++; ... }  // 依赖执行顺序

@Test
void test2() { assertEquals(2, counter); ... }
```

### 5.3 Mock 使用原则

**✅ 推荐**:
- Mock 外部依赖 (Mapper、Service、Util)
- 不 Mock 被测类本身
- 不 Mock 值对象 (Entity、DTO)

**❌ 避免**:
```java
// 不要 Mock 被测试的类
@Mock
private ActivityService activityService;  // 错误！

@InjectMocks
private ActivityService service;  // 正确 - 注入依赖
```

### 5.4 异常测试

**✅ 推荐 (JUnit 5)**:
```java
@Test
@DisplayName("获取活动 - 不存在时抛出异常")
void getActivityById_NotFound() {
    when(activityMapper.selectById(1)).thenReturn(null);

    BusinessException exception = assertThrows(
        BusinessException.class, 
        () -> activityService.getActivityById(1)
    );
    
    assertEquals(ErrorCode.ACTIVITY_NOT_FOUND.getCode(), 
                 exception.getErrorCode().getCode());
}
```

**❌ 避免 (老式 try-catch)**:
```java
@Test
void testException() {
    try {
        service.doSomething();
        fail("应该抛出异常");  // 不推荐
    } catch (Exception e) {
        assertEquals("错误", e.getMessage());  // 冗长
    }
}
```

### 5.5 测试覆盖率目标

| 模块 | 目标覆盖率 | 优先级 |
|------|-----------|-------|
| **Service 层** | 80%+ | 🔴 高 |
| **Controller 层** | 70%+ | 🟡 中 |
| **Util 工具类** | 90%+ | 🔴 高 |
| **Entity/DTO** | 可跳过 | ⚪ 低 |

## 6. 运行测试与覆盖率

### 6.1 Maven 命令

```bash
# 运行所有测试
mvn test

# 运行单个测试类
mvn test -Dtest=ActivityServiceTest

# 运行单个测试方法
mvn test -Dtest=ActivityServiceTest#createActivity_Success

# 运行并生成覆盖率报告 (需要 JaCoCo 插件)
mvn clean test jacoco:report

# 查看覆盖率报告 (HTML)
# 打开 target/site/jacoco/index.html
```

### 6.2 IntelliJ IDEA

1. **运行单个测试**: 点击测试方法旁的绿色三角
2. **运行整个类**: 点击类名旁的绿色三角
3. **运行带覆盖率**: 右键 → Run 'Test' with Coverage
4. **重新运行失败的测试**: Ctrl+Shift+R

### 6.3 添加 JaCoCo 覆盖率插件

在 `pom.xml` 中添加:

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.jacoco</groupId>
            <artifactId>jacoco-maven-plugin</artifactId>
            <version>0.8.11</version>
            <executions>
                <execution>
                    <goals>
                        <goal>prepare-agent</goal>
                    </goals>
                </execution>
                <execution>
                    <id>report</id>
                    <phase>test</phase>
                    <goals>
                        <goal>report</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>
```

## 📚 附录

### A. 测试检查清单

在提交测试前检查:

- [ ] 测试类名以 `Test` 结尾 (如 `ActivityServiceTest`)
- [ ] 测试方法遵循 `方法名_场景_预期` 命名
- [ ] 每个测试都是独立的
- [ ] 没有硬编码的魔法数字
- [ ] 异常测试验证了错误码
- [ ] Mock 行为覆盖了所有分支
- [ ] 测试通过了且覆盖率达标

### B. 常用 Mockito 速查

```java
// 基础
when(mock.method()).thenReturn(value);
when(mock.method(any())).thenReturn(value);

// void 方法
doNothing().when(mock).voidMethod();
doThrow(Exception.class).when(mock).voidMethod();

// 验证
verify(mock).method();
verify(mock, times(2)).method();
verify(mock, never()).method();
verify(mock, atLeastOnce()).method();

// 参数匹配
any(), anyString(), anyInt(), anyList()
eq(value), isNull(), notNull()
```
