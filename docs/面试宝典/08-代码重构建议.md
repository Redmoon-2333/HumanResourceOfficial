## 1. 构造器注入重构

### 问题说明

**当前代码** (字段注入，不推荐):

```java
@Service
public class ActivityService {
    
    @Autowired  // 字段注入
    private ActivityMapper activityMapper;
    
    @Autowired
    private ActivityImageMapper activityImageMapper;
    
    @Autowired
    private LocalFileUtil localFileUtil;
}
```

**问题**:
1. 依赖隐藏，难以发现循环依赖
2. 单元测试困难 (必须启动 Spring 容器)
3. 字段可能为 `null` ( Spring 容器未完全初始化时)
4. 违反依赖注入的显式原则

### 重构方案

**推荐写法** (构造器注入):

```java
@Service
@RequiredArgsConstructor  // Lombok 生成构造器
public class ActivityService {
    
    private final ActivityMapper activityMapper;
    private final ActivityImageMapper activityImageMapper;
    private final LocalFileUtil localFileUtil;
    
    // Lombok @RequiredArgsConstructor 自动生成:
    // public ActivityService(ActivityMapper activityMapper, 
    //                        ActivityImageMapper activityImageMapper, 
    //                        LocalFileUtil localFileUtil) {
    //     this.activityMapper = activityMapper;
    //     this.activityImageMapper = activityImageMapper;
    //     this.localFileUtil = localFileUtil;
    // }
}
```

### 需要重构的文件清单

| 文件 | 当前注入方式 | 依赖数量 | 优先级 |
|------|------------|---------|--------|
| `ActivityService.java` | 字段注入 | 3 | P1 |

## 2. 事务注解补充

### 问题说明

**当前代码** (缺少 `@Transactional`):

```java
@Service
public class AuthService {
    
    /**
     * 用户使用激活码注册
     * 涉及两张表的写入，需要事务保证
     */
    // ❌ 缺少 @Transactional
    public RegisterResponse registerByActivationCode(RegisterRequest request) {
        // 1. 验证激活码
        ActivationCode code = activationCodeMapper.findByCode(request.getCode());
        
        // 2. 创建用户 (失败则激活码状态不一致)
        User user = new User();
        userMapper.insert(user);
        
        // 3. 标记激活码已使用
        code.setStatus(ActivationStatus.已使用);
        activationCodeMapper.updateById(code);
        
        return new RegisterResponse(user.getUserId(), "注册成功");
    }
}
```

**风险**: 如果第 2 步失败，激活码状态不会回滚，导致数据不一致。

### 重构方案

**添加事务注解**:

```java
@Service
public class AuthService {
    
    /**
     * 用户使用激活码注册
     * 涉及两张表的写入，需要事务保证
     */
    @Transactional(rollbackFor = Exception.class)  // ✅ 添加事务
    public RegisterResponse registerByActivationCode(RegisterRequest request) {
        // 1. 验证激活码
        ActivationCode code = activationCodeMapper.findByCode(request.getCode());
        if (code == null) {
            throw new BusinessException(ErrorCode.INVALID_ACTIVATION_CODE);
        }
        
        // 2. 创建用户
        User user = new User();
        // ... 设置属性
        userMapper.insert(user);
        
        // 3. 标记激活码已使用
        code.setStatus(ActivationStatus.已使用);
        activationCodeMapper.updateById(code);
        
        return new RegisterResponse(user.getUserId(), "注册成功");
    }
}
```

### 需要添加事务的方法清单

| 文件 | 方法 | 涉及表 | 优先级 |
|------|------|--------|--------|
| `AuthService.java` | `registerByActivationCode` | user, activation_code | P0 |
| `AuthService.java` | `activateCode` | activation_code | P0 |
| `UserService.java` | `deleteActivationCode` | activation_code | P0 |
| `ActivityService.java` | `deleteActivity` | activity, activity_image | P0 |

## 3. 异常处理规范

### 问题说明

**当前代码** (异常处理不规范):

```java
public Activity getActivityById(Integer activityId) {
    try {
        Activity activity = activityMapper.selectById(activityId);
        if (activity == null) {
            throw new BusinessException(ErrorCode.ACTIVITY_NOT_FOUND);
        }
        return activity;
    } catch (Exception e) {
        // ❌ 捕获所有异常，丢失原始堆栈
        logger.error("获取活动详情时发生异常:{}", e.getMessage(), e);
        throw new BusinessException(ErrorCode.ACTIVITY_DETAIL_FAILED);
    }
}
```

**问题**:
1. `catch (Exception e)` 过于宽泛
2. 重新抛出时未保留原始异常原因
3. 错误信息不够详细

### 重构方案

**规范写法**:

```java
public Activity getActivityById(Integer activityId) {
    logger.info("根据 ID 获取活动详情：ID={}", activityId);
    
    try {
        Activity activity = activityMapper.selectById(activityId);
        
        if (activity == null) {
            logger.warn("未找到指定活动：ID={}", activityId);
            throw new BusinessException(ErrorCode.ACTIVITY_NOT_FOUND);
        }
        
        logger.info("成功获取活动详情：ID={}", activityId);
        return activity;
        
    } catch (BusinessException e) {
        // 业务异常直接抛出，不记录 error 日志
        throw e;
        
    } catch (DataAccessException e) {
        // 数据库访问异常，记录详细日志
        logger.error("数据库访问失败：ID={}, error={}", activityId, e.getMessage(), e);
        throw new BusinessException(ErrorCode.DATABASE_ERROR, e);
        
    } catch (Exception e) {
        // 其他未知异常
        logger.error("获取活动详情时发生未知异常：ID={}, error={}", 
                    activityId, e.getMessage(), e);
        throw new BusinessException(ErrorCode.SYSTEM_ERROR, e);
    }
}
```

**ErrorCode 补充** (增加数据库错误):

```java
public enum ErrorCode {
    // 数据库相关 3000-3999
    DATABASE_ERROR(3001, "数据库访问失败"),
    DATABASE_TIMEOUT(3002, "数据库操作超时"),
    
    // ... 其他错误码
}
```

### 重构方案

**方案 1: 提取工具方法**

```java
@Service
public class ActivityService {
    
    /**
     * 获取活动，不存在则抛出异常
     */
    private Activity getActivityOrThrow(Integer activityId) {
        Activity activity = activityMapper.selectById(activityId);
        if (activity == null) {
            throw new BusinessException(ErrorCode.ACTIVITY_NOT_FOUND);
        }
        return activity;
    }
    
    /**
     * 包装数据库操作，统一异常处理
     */
    private <T> T wrapDbOperation(Supplier<T> operation, ErrorCode errorCode) {
        try {
            return operation.get();
        } catch (DataAccessException e) {
            logger.error("数据库操作失败：{}", errorCode.getMessage(), e);
            throw new BusinessException(errorCode, e);
        }
    }
    
    public Activity updateActivity(Integer activityId, Activity activityDetails) {
        return wrapDbOperation(() -> {
            Activity existingActivity = getActivityOrThrow(activityId);
            
            existingActivity.setActivityName(activityDetails.getActivityName());
            existingActivity.setBackground(activityDetails.getBackground());
            // ...
            
            activityMapper.updateById(existingActivity);
            logger.info("活动更新成功：ID={}", activityId);
            return existingActivity;
        }, ErrorCode.ACTIVITY_UPDATE_FAILED);
    }
}
```

**方案 2: 使用 BaseRepository**

```java
// base/RepositorySupport.java
public abstract class RepositorySupport {
    
    protected <T> T findByIdOrThrow(BaseMapper<T> mapper, Integer id, ErrorCode errorCode) {
        T entity = mapper.selectById(id);
        if (entity == null) {
            throw new BusinessException(errorCode);
        }
        return entity;
    }
}

// ActivityService.java
@Service
public class ActivityService extends RepositorySupport {
    
    public Activity updateActivity(Integer activityId, Activity activityDetails) {
        Activity existingActivity = findByIdOrThrow(
            activityMapper, 
            activityId, 
            ErrorCode.ACTIVITY_NOT_FOUND
        );
        // ...
    }
}
```

## 5. N+1 查询优化

### 问题说明

**当前代码** ( UserService.java 中已优化，但其他地方可能存在):

```java
// ❌ N+1 查询示例
public List<ActivityDTO> getActivitiesWithImages() {
    List<Activity> activities = activityMapper.selectList(null);
    
    List<ActivityDTO> result = new ArrayList<>();
    for (Activity activity : activities) {
        ActivityDTO dto = new ActivityDTO();
        dto.setActivity(activity);
        
        // N+1 问题：每次循环查一次数据库
        List<ActivityImage> images = activityImageMapper.findByActivityId(activity.getActivityId());
        dto.setImages(images);
        
        result.add(dto);
    }
    return result;
}
```

### 重构方案

**批量查询优化**:

```java
// ✅ 优化后
public List<ActivityDTO> getActivitiesWithImages() {
    // 1. 查询所有活动
    List<Activity> activities = activityMapper.selectList(null);
    
    // 2. 收集所有活动 ID
    List<Integer> activityIds = activities.stream()
        .map(Activity::getActivityId)
        .collect(Collectors.toList());
    
    // 3. 批量查询图片 (一次查询)
    List<ActivityImage> allImages = activityImageMapper.selectBatchIds(activityIds);
    
    // 4. 按活动 ID 分组
    Map<Integer, List<ActivityImage>> imagesByActivityId = allImages.stream()
        .collect(Collectors.groupingBy(ActivityImage::getActivityId));
    
    // 5. 构建 DTO
    return activities.stream().map(activity -> {
        ActivityDTO dto = new ActivityDTO();
        dto.setActivity(activity);
        dto.setImages(imagesByActivityId.getOrDefault(activity.getActivityId(), Collections.emptyList()));
        return dto;
    }).collect(Collectors.toList());
}
```

```java
// SecurityConfig.java
@Configuration
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.cors(cors -> cors.configurationSource(corsConfigurationSource()));
        // ...
        return http.build();
    }
    
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();
        // ❌ 硬编码允许的源
        config.setAllowedOrigins(Arrays.asList("http://localhost:3000"));
        // ...
    }
}
```

### 重构方案

**配置外部化**:

```yaml
# application.yml
app:
  cors:
    allowed-origins: ${CORS_ALLOWED_ORIGINS:http://localhost:3000}
    allowed-methods: GET,POST,PUT,DELETE,OPTIONS
    allowed-headers: "*"
    allow-credentials: true
    max-age: 3600
```

```java
// SecurityConfig.java
@Configuration
@ConfigurationProperties(prefix = "app.cors")
@Data
public class SecurityConfig {
    
    private List<String> allowedOrigins;
    private String allowedMethods;
    private String allowedHeaders;
    private Boolean allowCredentials;
    private Long maxAge;
    
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowedOrigins(allowedOrigins);  // ✅ 从配置读取
        config.setAllowedMethods(Arrays.asList(allowedMethods.split(",")));
        config.setAllowedHeaders(Arrays.asList(allowedHeaders.split(",")));
        config.setAllowCredentials(allowCredentials);
        config.setMaxAge(maxAge);
        // ...
    }
}
```

## 8. 参数校验增强

### 问题说明

**当前代码** (参数校验不完整):

```java
@PostMapping("/activity")
public ApiResponse<Activity> createActivity(@RequestBody Activity activity) {
    // ❌ 缺少 @Validated 校验
    // ❌ 手动校验容易遗漏

        return ApiResponse.error("活动名称不能为空");
    }
    // ...
```

### 重构方案

**使用 Bean Validation**:

```java
// dto/ActivityCreateRequest.java
@Data
public class ActivityCreateRequest {
    
    @NotBlank(message = "活动名称不能为空")
    @Size(max = 100, message = "活动名称不能超过 100 个字符")
    private String activityName;
    
    @Size(max = 500, message = "活动背景不能超过 500 个字符")
    private String background;
    
    @Size(max = 500, message = "活动意义不能超过 500 个字符")
    private String significance;
    
    @Size(max = 500, message = "活动目的不能超过 500 个字符")
    private String purpose;
    
    @Size(max = 1000, message = "活动流程不能超过 1000 个字符")
    private String process;
}
```

```java
// Controller 使用
@PostMapping("/activity")
public ApiResponse<Activity> createActivity(
        @RequestBody @Validated ActivityCreateRequest request) {
    // ✅ 参数校验自动完成，不通过直接返回 400
    
    Activity activity = new Activity();
    BeanUtils.copyProperties(request, activity);
    
    Activity created = activityService.createActivity(activity);
    return ApiResponse.success(created);
}
```

## 重构检查清单

完成重构后，请确认以下项目:

- [ ] 所有涉及多表操作的方法都添加了 `@Transactional`
- [ ] 所有 `@Service` 类使用构造器注入
- [ ] 所有 `@RestController` 类使用构造器注入
- [ ] 所有 `catch (Exception e)` 都记录了完整堆栈
- [ ] 所有业务异常使用 `BusinessException`
- [ ] 所有日志不包含敏感信息 (密码、token)
- [ ] 所有 Controller 参数有 `@Validated` 校验
- [ ] 所有配置项都从配置文件读取
- [ ] 所有单元测试通过
- [ ] 代码覆盖率不低于 80%
