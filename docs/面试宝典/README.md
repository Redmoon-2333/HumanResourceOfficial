## 🎯 面试技巧

### 回答问题的 STAR 法则

| 要素 | 含义 | 示例 |
|------|------|------|
| **S**ituation | 情境 | "在我的项目中，登录接口面临高频访问..." |
| **T**ask | 任务 | "我需要实现限流功能，防止接口滥用..." |
| **A**ction | 行动 | "我采用了滑动窗口算法 + Lua 脚本..." |
| **R**esult | 结果 | "成功将接口响应时间从 100ms 降低到 30ms..." |

### 回答技术问题的结构

```
1. 先说结论 (直接回答问题)
2. 展开原理 (深入解释技术细节)
3. 结合项目 (说明在项目中如何应用)
4. 总结收获 (体现学习和思考)
```

**示例**: 什么是 Redis 分布式锁？

```
[结论] Redis 分布式锁是使用 Redis 实现的跨进程、跨机器的锁机制。

[原理] 核心命令是 SET key value NX EX，NX 表示不存在时才设置，EX 设置过期时间。
      本项目使用 Redisson 实现，支持看门狗自动续期、可重入等特性。

[项目] 在我的项目中，创建活动接口使用了@DistributedLock 注解，
       防止并发创建同名活动，保证数据一致性。

[收获] 通过这个项目，我深入理解了分布式锁的实现原理，
       以及 Redisson 看门狗机制的设计思想。
```

## 🌟 项目亮点总结

面试时，可以这样介绍你的项目:

```
我这个人⼒资源官网项目采用了 Spring Boot 3.3.5 作为核心框架，
整合了 Redis、Spring AI、MyBatis-Plus 等技术栈。

项目的技术亮点包括:

1. 完整的缓存体系
   - 使用@Cacheable 实现活动列表缓存，查询性能提升 25 倍
   - 布隆过滤器防止缓存穿透，互斥锁防止缓存击穿
   - Cache Aside 模式保证缓存与数据库一致性

2. 注解 + 切面的架构设计
   - 自研@RateLimit 注解，基于滑动窗口 + Lua 脚本实现限流
   - 自研@DistributedLock 注解，基于 Redisson 实现分布式锁
   - 通过 AOP 实现横切关注点，业务代码简洁清晰

3. AI 功能集成
   - 使用 Spring AI 1.0.0 集成大模型对话
   - 实现 RAG 检索增强，基于知识库回答问题
   - 流式响应 + 对话记忆管理，提升用户体验

4. 安全机制完善
   - JWT 认证 + RBAC 权限控制
   - bcrypt 密码加密
   - SQL 注入/XSS 攻击防护

通过这个项目，我不仅掌握了 Spring Boot 全栈开发能力，
更重要的是培养了分层架构设计思维和工程质量意识。
```
