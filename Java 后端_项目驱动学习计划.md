# Java 后端 - 项目驱动式学习计划（3 个月）

> 核心理念：**以项目升级为载体，缺什么学什么**，避免理论与实践脱节
> 目标：3 个月内将现有项目改造为**企业级微服务架构**，同时掌握核心技能

---

## 📊 你当前的项目资产

| 项目 | 技术栈 | 可升级点 |
|-----|--------|----------|
| 人力资源官网 | Spring Boot + Redis + Spring AI | 拆微服务、加 MQ、引入 Sentinel |
| 博远官网 | Spring Boot + MySQL + Redis | 拆微服务、JWT 改造、加监控 |

---

## 📅 三阶段升级路线

```
第 1 月：单体优化 → 第 2 月：微服务拆分 → 第 3 月：生产级加固
```

---

## 第一阶段：单体优化（第 1-4 周）

**目标**：在不改变架构的前提下，优化现有代码质量，为微服务化做准备

### 第 1 周：代码重构 + 单元测试覆盖

**升级任务**（选人力资源官网）
- [ ] 用 IntelliJ 分析代码异味（Code Smell）
- [ ] 提取重复代码为工具方法
- [ ] Controller → Service → Mapper 分层梳理
- [ ] 为核心 Service 编写 JUnit5 测试（覆盖率>60%）

**学习内容**
- 设计模式：策略模式（用于多种登录方式）、模板方法模式（用于 CRUD）
- 单元测试：Mockito  mock 依赖、@SpringBootTest 集成测试

**产出物**
- 重构后的代码（提交 Git）
- 单元测试用例 ≥ 20 个
- 博客：《Spring Boot 项目单元测试实战》

---

### 第 2 周：Redis 缓存深度应用

**升级任务**
- [ ] 分析高频查询接口（用 Actuator 或手动日志）
- [ ] 为活动列表、用户信息添加 Redis 缓存
- [ ] 实现 Cache Aside 模式（先查缓存→ miss 查 DB→回写缓存）
- [ ] 处理缓存穿透（布隆过滤器）、缓存击穿（互斥锁）

**学习内容**
- Redis 数据结构选型：String vs Hash vs ZSet
- 缓存一致性策略：延迟双删 vs Canal 监听 binlog
- Redisson 分布式锁使用

**产出物**
- 缓存优化代码
- 压测报告（优化前后 QPS 对比）
- 博客：《Redis 缓存实战：从穿透到雪崩》

---

### 第 3 周：异步解耦（引入消息队列）

**升级任务**
- [ ] 识别可异步场景：邮件发送、短信通知、文件解析
- [ ] 引入 RabbitMQ/RocketMQ（本地 Docker 部署）
- [ ] 改造邮件发送为异步
- [ ] 实现消息可靠性（确认机制 + 重试 + 死信队列）

**学习内容**
- MQ 核心概念：Exchange、Queue、Routing Key
- 消息丢失/重复/积压处理
- 最终一致性 vs 强一致性

**产出物**
- 异步邮件/短信模块
- 博客：《消息队列在 Spring Boot 中的实战》

---

### 第 4 周：接口限流与监控

**升级任务**
- [ ] 引入 Sentinel 或自定义限流（令牌桶算法）
- [ ] 为高频接口（如验证码发送）配置限流规则
- [ ] 集成 Spring Boot Actuator + Prometheus
- [ ] 配置 Grafana 监控看板

**学习内容**
- 限流算法：固定窗口、滑动窗口、令牌桶、漏桶
- 监控指标：QPS、响应时间、错误率、JVM 内存

**产出物**
- 限流配置代码
- Grafana 监控看板截图
- 博客：《Spring Boot 接口限流与监控实战》

## 🎯 关键建议

1. **每周节奏**：
   - 周一～周三：学习理论知识（视频/文档）
   - 周四～周六：动手改造代码
   - 周日：写博客总结 + 准备下周

2. **遇到卡点怎么办**：
   - 优先查官方文档
   - 其次查 Stack Overflow / GitHub Issues
   - 不要超过 2 小时不解决就跳过/求助

3. **时间不够怎么办**：
   - 优先完成带 [核心] 标记的任务
   - 博客可以延后，但代码一定要写
